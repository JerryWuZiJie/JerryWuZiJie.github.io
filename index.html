my first post using hexo framework for blog

Hi, this is Jerry (Zijie) Wu! I decided to start writing blogs. And after a quick research, I found that hexo seems to fit my needs. It can be depolyed on GitHub so I don’t have to rent a server. I’ve also consider using LinkedIn as a blog platform, but seems like not much people are doing that and it doesn’t look “geeky.”

I will mainly use the blog to write about things related to my major (Electrical and Computer Engineering) and hope to share my knowledge and experience with others (if anyone ever visit).

Setup Hexo

In the following I will briefly talk about how to setup Hexo and deploy it on GitHub and how to use it.

Install

  1. install node.js
  2. open terminal
    1. npm install hexo-cli -g to install hexo
    2. mkdirdirectory to create a new directory for a new Hexo project
    3. cddirectory to enter the new directory
    4. hexo init to initialize a new Hexo project
    5. hexo server to start a server on local host (hexo s for short)
    6. hexo newpost_name to create a new post (hexo n for short)
    7. hexo clean to clean the static files
    8. hexo generate to generate new static files
    9. npm install hexo-deployer-git --save to install the git deployer
  3. create new repository on GitHub, name must be githubusername.github.io
  4. in the _config.yml file, modify the deploy section:
  5. run hexo deploy in terminal to deploy to GitHub (hexo d for short)

Usage

  • command
    • hexo n "title": create a new post
    • hexo s: start a server in local host
    • hexo c: clean the static files
    • hexo g: generate the static files
    • hexo d: deploy to GitHub
  • adding image
    1. in _config.yml, change false to true in post_asset_folder: false
    2. use the syntax {% asset_img image.png Alt Text %} to add image if image.png is in the folder with the same name as the post
    3. [optional] install hexo-asset-link (npm i -s hexo-asset-link) to support Markdown image syntax (![Alt Text](image.png))
  • tags: you can have tags in the very top of the post using syntax tags: [tag1, tag2, tag3, ...]
  • categories: you can have categories in the very top of the post using following syntax
